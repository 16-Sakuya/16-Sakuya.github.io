实验二：代码编辑与管理

实验目标：

  • 在vscode中安装和配置Markdown相关插件，掌握代码编辑器插件的基本概念
  • 学习Markdown基础语法，使用VSCode完成Markdown文档的编辑和渲染任务
  • 构建一个静态站点并将其部署到GitHub Pages

安装插件

  • 打开vscode扩展，下载简体中文、markdown相关插件
    • 简中：方便我们看懂
    • markdown相关：
      • Markdown All in One：目录生成、快捷键支持、自动完成、列表自动延续等
      • Markdown Preview Enhanced：数学公式、流程图、导出为PDF/HTML等
      • Paste Image：直接粘贴剪贴板中的图片到Markdown文件

创建、编辑markdown文件

  1. 在项目文件夹中打开命令提示符，运行*hexo new “文章名”*创建一篇新文章
  2. 在./source/_posts文件夹下找到新创建的markdown文件，在vscode中打开编辑
  3. 保存关闭。重新hexo generate、hexo s查看界面

部署

    - 仓库名必须设为GitHub用户名.github.io
    - 选择公开，不创建README。创建仓库
  2. 在项目文件夹中打开终端，运行npm install hexo-deployer-git –save安装git部署插件

实验一：web开发环境

实验目标：

  • 安装并运行一个Web博客软件（hexo）
  • 了解和掌握Web开发环境的基础安装与配置（Node.js、npm、Git）
  • 理解静态博客的原理及特点

环境的基础安装与配置

Node.js、npm安装：官网安装

  • npm会自动配置

Git for Windows安装：官网安装

  • 通过*git config –global user.name “你的GitHub姓名”、git config –global user.email “你的GitHub邮箱”*配置全局用户信息

hexo-cli安装：在命令提示符窗口输入：npm install -g hexo-cli

验证安装：

在命令提示符中输依次入node -v、npm -v、git -v、hexo -v回车，若能查询到版本则说明安装成功

运行hexo

  1. 在存放博客的文件夹中打开终端，运行hexo init 你的项目名创建博客项目
  2. 在终端输入hexo generate重新生成静态页面，接着输入hexo s(hexo server)启动本地预览服务


    在浏览器中打开最后一行的网址即可预览页面

总结

  • 静态博客的原理：在本地将Markdown等源文件预编译为纯HTML页面
  • 静态博客的特点：加载速度快、安全性高、无需数据库
